# JSON Editor Guide

The JSON editor provides direct access to the underlying flowchart data structure, allowing for precise control and bulk operations.

## Accessing the JSON Editor

1. Open any flowchart at `/admin/flow/[id]`
2. Click **"Show Code"** button
3. Select the **"JSON"** tab

## Data Structure

### Flowchart Object

```json
{
  "nodes": [...],
  "edges": [...]
}
```

### Node Object

```json
{
  "id": "node-1234567890-abc123",
  "type": "default",
  "position": {
    "x": 200,
    "y": 100
  },
  "data": {
    "title": "Process Data",
    "content": "Transform input"
  }
}
```

**Properties:**
- `id` (string): Unique identifier
- `type` (string): `'input'`, `'default'`, or `'output'`
- `position` (object): X and Y coordinates
- `data` (object): Node content with `title` and `content`

### Edge Object

```json
{
  "id": "enode-123-node-456",
  "source": "node-123",
  "target": "node-456",
  "type": "animated"
}
```

**Properties:**
- `id` (string): Unique identifier
- `source` (string): Source node ID
- `target` (string): Target node ID
- `type` (string): Edge type (usually `'animated'`)

## Real-Time Sync

The JSON editor syncs in real-time with the visual flowchart:
- Edit JSON → Flowchart updates instantly
- Move nodes visually → JSON updates automatically

## Common Operations

### Add a Node

```json
{
  "nodes": [
    {
      "id": "new-node-1",
      "type": "default",
      "position": { "x": 300, "y": 200 },
      "data": {
        "title": "New Node",
        "content": "Description"
      }
    }
  ],
  "edges": []
}
```

### Connect Nodes

```json
{
  "nodes": [...],
  "edges": [
    {
      "id": "e1-2",
      "source": "node-1",
      "target": "node-2",
      "type": "animated"
    }
  ]
}
```

### Bulk Position Update

```json
{
  "nodes": [
    { "id": "1", "position": { "x": 100, "y": 100 }, ... },
    { "id": "2", "position": { "x": 300, "y": 100 }, ... },
    { "id": "3", "position": { "x": 500, "y": 100 }, ... }
  ]
}
```

### Delete Nodes

Simply remove the node object from the `nodes` array and any related edges.

## Tips

- Use a JSON formatter for better readability
- Validate JSON before saving to avoid errors
- Keep node IDs unique
- Ensure edge source/target IDs exist in nodes array

## Example: Complete Workflow

```json
{
  "nodes": [
    {
      "id": "start-1",
      "type": "input",
      "position": { "x": 100, "y": 200 },
      "data": {
        "title": "Start",
        "content": "Begin process"
      }
    },
    {
      "id": "process-1",
      "type": "default",
      "position": { "x": 300, "y": 200 },
      "data": {
        "title": "Process",
        "content": "Transform data"
      }
    },
    {
      "id": "end-1",
      "type": "output",
      "position": { "x": 500, "y": 200 },
      "data": {
        "title": "End",
        "content": "Complete"
      }
    }
  ],
  "edges": [
    {
      "id": "e-start-process",
      "source": "start-1",
      "target": "process-1",
      "type": "animated"
    },
    {
      "id": "e-process-end",
      "source": "process-1",
      "target": "end-1",
      "type": "animated"
    }
  ]
}
```
